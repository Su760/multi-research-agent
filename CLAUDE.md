## Running the Application

```bash
# CLI argument mode (runs once)
python3 main.py "your topic here"

# Interactive mode (loops until 'quit')
python3 main.py
```

Install dependencies:
```bash
pip install python-dotenv langchain-groq langchain-tavily langgraph
```

Set both keys in `.env` (do not commit this file):
```
GROQ_API_KEY=...
TAVILY_API_KEY=...
```

Reports are saved to `reports/<topic-slug>.md` after each run.

## Architecture

A sequential multi-agent research pipeline built with **LangGraph**. All code lives in `main.py`.

**Pipeline:** `Topic → Planner → Researcher → Synthesizer → Critic → Rewriter → Report`

**Shared state** (`ResearchState` TypedDict):
- `topic` → input research question
- `questions` → 5 sub-questions generated by Planner
- `research` → aggregated web search results
- `report` → current report (written by Synthesizer, rewritten by Rewriter)
- `critic_feedback` → 2-3 weak points identified by Critic

**Agents:**
- **Planner** — uses `ChatGroq(llama-3.3-70b-versatile)` to decompose the topic into 5 specific questions
- **Researcher** — runs `TavilySearch(max_results=3)` for each question; logs errors and continues on failure
- **Synthesizer** — writes a 3-4 paragraph first-draft report from the research
- **Critic** — reviews the draft and returns a numbered list of 2-3 weak points or gaps
- **Rewriter** — rewrites the report addressing the critic's feedback

The graph is linear (no branching). LangGraph compiles a `StateGraph` with edges: `planner → researcher → synthesizer → critic → rewriter → END`.

**External dependencies:**
- **Groq API** — LLM inference via `langchain-groq`
- **Tavily** — web search via `langchain-tavily`
